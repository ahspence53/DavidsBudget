Overview

This app is designed to help you plan ahead, not just track spending.

  • Day-before-payday focus — ensures your balance stays safe before income arrives
  • Forward projection — see up to 24 months ahead
  • What If testing — safely explore savings or spending changes
  • Local data only — nothing is stored online
  • Offline mode - you do not need an internet connection to access the app once it's built

Initial set up

  • Set your start date and opening balance
  • Create your categories (you must have categories to add a transaction)
  • Add your regular income salary/pension)
  • Add your regular expenses
  • The projection updates immediately.
If your bank shows £150 and you have £50 of transactions on that day, your opening balance should be £200.

Savings Pots

You can 'link' the home budget app to your separate Savings account(s) pots
Any expense with the Catergory of Savings will be assumed to be transferred to
a Savings account.
Start by entering opening balances (if any) for each pot, then Save Opening Balances

On the date of future Savings transactions, the current value of each pot will be updated

If you withdraw money from a Savings pot simply amend the opening balance of the pot

When you add a Transaction select the Category Savings and you will be prompted to select which
Savings Pot the Transaction will go into.
Note! The id and Description of each Savings Pot is 'hard coded' into the github file APP.JS
eg.
{
id: "carBudget",
name: "Savings Car Budget",
openingBalance: 0
}

Import/Export transaction

Use Export and Import JSON to backup and restore your transactions

  • Export Transactions - JSON Saves your transactions to a file (.JSON)
  • Import Transactions - JSON Imports your transactions from a file (.JSON)
  • Once you have added transactions, or when you make changes, you should backup

  • To rebuild your app, delete your browser cache, open the app in your browser (https://ahspence53.github.io/Budget/)
    • set the Start date and Opening balance, then from the Maintenance menu then select Import Transactions - JSON
Import Transactions - JSON restores;
- transactions
- categories
- savings pots
- savings pots opening balances

FAQ's

How to ensure the app ALWAYS works
After initially building the app by entering Start date and Opening balance
then adding Categories and Transactions you should make a Backup
see Import/Export Transactions
By doing this you can easily re-build the app with your data.

Why does my balance go negative?
Expenses exceed available funds before your next income.

Why do transactions disappear?
The app is forward-looking — past transactions are removed when you move the start date.

Why is What If lower than expected?
The buffer ensures your balance never drops below a safe level.

How do I ensure I don't lose my data?
You should regularly BACKUP, using Export Transactions - JSON
especially when you make any changes to your transactions.
Also, in Diary, click Backup to save the .JSON file to your local file system.

How do I 'Start' over?
Simply delete your browser cache.
Type the app name into your browser - https://ahspence53.github.io/Budget/
Enter Start Date and Opening Balance
then Import Transactions - JSON and select your .JSON backup file.
Finally Restore any Diary backup you have.

Do my Categories get re-created when I restore from backup?
Yes Categories are saved when you do a backup (Export Transactions - JSON)
They will be re-created when the data is loaded into the App from the JSON file.
